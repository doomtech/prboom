Installation on Linux and UNIX Systems

This page guides you through the first steps of getting PrBoom, namely downloading the right PrBoom package for your system, and getting the necessary libraries.

Please see the most appropriate instructions depending on your system distribution:

  • Linux/x86 RPM based distros (Redhat, Mandrake, ...)
  • Debian (and derivatives)
  • FreeBSD (may be appropriate for other BSDs)
  • Generic UNIX (anything else)

If you want music support, see the music section.

Compiling with OpenGL support

I didn't write the OpenGL stuff, nor do I have a good 3D card to test with, so I don't use this nor know much about it. That said, it did work for me last time I tried it.

The RPM distribution of PrBoom doesn't support this, so you must compile from source. Follow the instructions below relevant to your distribution, but add the parameter --enable-gl when you run configure. You'll need Mesa installed.

I don't suggest using the OpenGL stuff unless you're already familiar with Doom and PrBoom.

We got reports that it works with XFree 4.x

Linux/x86 RPM based distributions

You need to download the following things:

  • The main SDL library, SDL-1.1.4-1.i386.rpm (or a later version), from the SDL 1.1 download page.
  • smpeg, from smpeg-0.3.5-1.i386.rpm or later from the SMPEG homepage.
  • The SDL_mixer library, SDL_mixer-1.0.5-1.i386.rpm (or later), from the SDL_mixer homepage.
  • The SDL_net library. There is no official package of this so I made my own, SDL_net-1.1.1.i386.rpm, available from here.
  • PrBoom itself, prboom-2.1.0-0.i386.rpm, from the PrBoom download page.

In total that's about 850k to download

Exception: Mandrake ships with all this stuff on the CDs. smpeg (and timidity) on the Mandrake CDs are fine, but the versions of the SDL libraries on the Mandrake CDs are too old and buggy to use with PrBoom. Don't try it - I have, they're no good. Get new ones from the URLs above.

To install these, you use rpm. E.g. if you downloaded all of the above RPMs into one directory, cd to that directory and run rpm -ivh *.rpm. You can then run PrBoom as /usr/games/prboom.

Compiling from source

If you want to compile from source, then you need all of the above packages, and also SDL-devel-1.1.4-1.i386.rpm (from the SDL 1.1 download page) and SDL_mixer-devel-1.0.5-1.i386.rpm (again from the SDL_mixer homepage).

Debian

Debian unstable contains packages of SDL and SDL_mixer. First, install the packages libsdl1.1, libsdl1.1-dev, libsdl-mixer1.0, libsdl-mixer-dev, smpeg. You also need the SDL_net library, which is not yet packaged for Debian; get the source from the SDL_net homepage. Get the prboom source code prboom-2.1.0.tar.gz from our download page, extract, and then follow the included INSTALL instructions.

FreeBSD

I've tested on FreeBSD 4.1-STABLE. The ports tree contains packages of most of the stuff you need. Make the following ports:

  • devel/sdl-devel
  • audio/sdl_mixer (requires audio/smpeg)

    • You also need the SDL_net library, which you can get from it's homepage. Note that because the sdl-devel port installs in a strange directory, you'll need to ln -s ../SDL/SDL_net.h /usr/local/include/SDL11/SDL_net.h or something similar.

    Finally, get prboom-2.1.0.tar.gz the PrBoom source, extract, and follow the included INSTALL instructions.

    Generic UNIX

    You need to get and install the following libraries:

    • SDL 1.1.x, from the SDL 1.1 download page.
    • SDL_mixer, from its home page. This also requires the following:
      • SMPEG, from the SMPEG homepage.

      I don't know about these, look 'em up yourself :-).

    • SDL_net, from its home page.

    Then get prboom-2.1.0.tar.gz the PrBoom source, extract, and follow the included INSTALL instructions.

    Music

    The SDL_mixer library used by PrBoom supports software MIDI music synthesis. If you want to hear the Doom music, you need a set of Timidity instrument patches. Debian includes a package timidity-patches which provides these. People with RPM based systems can look for a package. Everyone else, do a web search for timidity patch sets, there are plenty around.

    These patch sets are a large download (>5megs).

    SDL_mixer does not currently support hardware MIDI synthesis. We plan to nag the SDL developers about this. If you want it, you nag them too :-).
